Description of the module

Programming Challenge 2 is a Computer Science and Engineering module which is studied by the 2nd year students.
The basic outcomes of this module is to understand the concepts of:

  • Networking
  • Software Engineering
  • Databases
  • Graphics
  • Artificial Intelligence
  • Web Developing

As the semester project, we are to develop the client side of a tank game. This is a two member group project and at the end of the semester, there will be a competition among the games developed

Game Information

About The Game

The main objective of this game is to play as a tank to accumulate points while taking care of its own survival

There are three types of blocks available in the gaming environment

  • Brick Wall - A hit will convert the wall into a blank cell
  • Stone Wall - Cannot be destroyed
  • Water - Sudden death if player moves to this cell
  • Blank cell - Player is free to move in these.

There are two ways a player can acquire points in the game

  • Breaking Bricks - When a player hits a shot to a brick a constant amount of points will be added to the player
  • Collecting coins - There are two types of coins that are there in the game.
    • Random treasures - Can collect tem by moving to the cell when the appear
    • Spoils of war - When a player destroyes another tank, 25%t of the destroyed players coin pile will be added to the player.

Life Packs will appear randomly as coin piles in te grid. The method to take a life pack is similar to a coin pile. A life pack will increase the health by 20% of the initial health. It is possible for a player to have more health than what he had in the start.

Rules

  • A timer will be started when the first player joins the game. After the 5th player joins or the expiration the game begins.
  • The game will be played in a 20x20 grid
  • The client can issue a command once a second
  • If hit an obstacle or being shot at, the health of the tank will be reduced
  • A bullet hit will reduce the player's health by 10%. A kill will drop a coin pile on the grid. Collection of that pile is similar to a random pile
  • If a player hit a brick, a fixed number of points will be deduced from the player.
  • The game is ended when all tanks are destroyed, when the time limit of the game is reached or all the coin piles are disappeared from the game.

Client

Game Client Details

We are given the server side of the game and we are developing the Client side of the game.
  • Technologies we used:
    • C# Language - To develop the Client side of the game
    • XNA Gaming Studio - To develop the Client graphical User Interface of the game
  • Thechniques we used:
    • Minimax Algorithm - To develop the searching of the AI and to predefine the next move
    • Game Protocol - The way sever communicates with the client. Here the server sends different Character Strings to communicate with the Client.
      • Example :- S:P{num}:{player location x},{player location y}:{Direction}# - This message is sent when the server accepts the request to join.
